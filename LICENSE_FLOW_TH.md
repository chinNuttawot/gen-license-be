# 📝 ระบบการออกและบันทึก License (License Generation Flow)

เอกสารนี้อธิบายกระบวนการทำงานของระบบหลังบ้าน (Backend) ในการสร้าง License และการบันทึกข้อมูลลงฐานข้อมูล (Database) เพื่อป้องกันการสร้างข้อมูลซ้ำซ้อนและรักษาความปลอดภัยของ Token

---

## 🏗️ สถาปัตยกรรม 2 ขั้นตอน (2-Step Process)

ระบบได้ถูกออกแบบให้แยกการทำงานออกเป็น 2 ขั้นตอนหลัก (ผ่าน 2 API Endpoint) เพื่อลดปัญหาข้อมูลขยะใน Database กรณีที่ผู้ใช้งานเพียงแค่ต้องการทดลอง "Preview" ข้อมูลเฉยๆ

### 1. ขั้นตอนการสร้าง (Generate Phase)
**Endpoint:** `POST /api/license/generate`

- **หน้าที่:** สร้าง Token ที่เซ็นด้วย **Ed25519 (Private Key)** 
- **การทำงาน:**
  - ตรวจสอบความถูกต้องของข้อมูล (Validate Payload)
  - ระบบสร้างเวลา `issuedAt` (ระดับมิลลิวินาที) แนบไปเป็น Metadata
  - สร้าง Token ตามจำนวนที่ขอ (Account-based) หรือตามจำนวน HWID (Station-based)
  - **ไม่มีการเชื่อมต่อหรือบันทึกลง Database ในขั้นตอนนี้** เพื่อป้องกันข้อมูลขยะ
- **ผลลัพธ์:** ส่ง Array ของ Token และ Metadata (ที่มี `issuedAt`) กลับไปให้ Frontend แสดงผล

### 2. ขั้นตอนการส่งออกและบันทึก (Export & Save Phase)
**Endpoint:** `POST /api/license/export`

- **หน้าที่:** เข้ารหัสไฟล์ `.aglic` ดาวน์โหลด และ **บันทึกลง Database**
- **การทำงาน:**
  - รับข้อมูล `tokens` และ `meta` กลับมาจาก Frontend
  - สร้างไฟล์ดาวน์โหลดด้วยการเข้ารหัส `AES-256-GCM` ด้วยรหัส `BUNDLE_KEY`
  - **บันทึกข้อมูลลงฐานข้อมูล (Database Save):**
    - ตรวจสอบว่ามีข้อมูล `Company` หรือยัง ถ้าไม่มีให้สร้างใหม่
    - เข้ารหัสตัว License Token ด้วยฟังก์ชัน `encryptToken` (เพื่อความปลอดภัยระดับสูง ไม่ให้ Token รั่วไหลแม้ Database โดนแฮก)
    - **ระบบป้องกันการบันทึกซ้ำ (Anti-Spam / Duplicate Check):** 
      - นำเวลา `meta.issuedAt` มาตั้งค่าเป็นเวลา `generatedAt` ในฐานข้อมูล
      - ทำการ Query เช็คก่อนว่าในตารางมีแถวของบริษัทนี้ ที่เวลา `generatedAt` ตรงกันเป๊ะๆ หรือไม่
      - หากพบว่า **มีแล้ว** -> จะข้ามการบันทึก DB และส่งไฟล์ให้ดาวน์โหลดอย่างเดียว (กรณีที่ผู้ใช้กดปุ่ม Export ซ้ำๆ)
      - หาก **ยังไม่มี** -> ทำการ INSERT ข้อมูลของ Token ทุกตัวลง Database (แยกแถวกัน) 
- **ผลลัพธ์:** ส่งไฟล์ `.aglic` (application/octet-stream) กลับไปให้เบราว์เซอร์ดาวน์โหลด

---

## 💾 โครงสร้างฐานข้อมูล (TypeORM)

- **`companies`**: เก็บข้อมูลชื่อบริษัท
- **`license_records`**: 
  - `companyId` (Foreign Key เชื่อมกับบริษัท)
  - `licenseType` (ประเภท เช่น account-based / station-based)
  - `quantity` (จำนวนที่ใช้งาน)
  - `hwid` (Hardware ID กรณีผูกติดเครื่อง)
  - `validUntil` (วันหมดอายุ)
  - `generatedAt` (เวลาที่ถูกสุ่มออกตอนกดปุ่ม Generate นำมาจาก `issuedAt`)
  - `encryptedToken` (ตัว Token จริงๆ ที่นำไปเข้ารหัสอีกชั้นก่อนเซฟลงตาราง)

---

## 🔒 เรื่องของความปลอดภัย (Security)

1. **Digital Signature:** ตัว License ภายในเป็นการเซ็นด้วย Private Key (Ed25519) ฝั่ง Client หรือโปรแกรมสามารถตรวจสอบได้ว่าของแท้หรือไม่ด้วย Public Key (ไม่ต้องใช้เน็ต)
2. **File Encryption:** ไฟล์ที่ส่งออกไป `.aglic` ถูกเข้ารหัสทั้งก้อนด้วย `AES-256-GCM` ต้องใช้ `BUNDLE_KEY` ในการเปิดอ่าน (เผื่อคนแอบดักจับไฟล์ไประหว่างทาง)
3. **DB Token Encryption:** ข้อมูล Token จริงๆ ถูกซ่อนรหัสก่อนลงฐานข้อมูล เพื่อป้องกันเคสที่ Database รั่วไหลแล้วมีคนก๊อปปี้ Token ไปใช้งานตรงๆ
